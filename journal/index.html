Entry #2

Febuary 4th, 2025

Have you ever gotten annoyed about overlays? Overlays are the text windows that appear on top of the main page of a website. They make it so that users can’t interact with the main page until you exit the overlay which many people find frustrating especially since they often get overused.

In this blog post I will be going over some key points from this Medium Article by Naema Baskanderi called, “Best Practices for Modals/Overlays/Dialog Windows.”

Modals are often used as a way to grab the users attention by interrupting their current task and switching to a new layout. This will then be used to show additional information related in some way to what is on the main page.

Modals can include different features. There is always an Escape Hatch which allows users to close the modal. Beyond the escape hatch the content of modals vary, for example, they can include the following: descriptive title, body text, image, buttons, and focus design effects.

Some main things to consider when designing modals are: Don’t overuse them, make sure they are accessible, and consider how they work on mobile format.

I hope you enjoyed reading my takeaways from this article about Modals!

The 20 practices are as follows: 1. Minimizing the Input Fields, 2. One Column Layout, 3. Multiple Steps, 4. Ask the Easy Questions First, 5. Distinguish Optional and Required Fields, 6. Never use a Reset or Clear Button, 20. Accessibility.

When reading through this list, I realized that forms have a lot of complexity and variety of inputs that I had not previously considered. A lot of the recommendations that the author gives are based on user experience. We want to make our forms as easy and straightforward as possible.

Going into specifics, I noticed some of the authors advice had a degree of preference. He recommends using radio buttons instead of drop downs whenever your lists of options is pretty short. While I agree that when you have minimal options radio buttons are more user friendly, I think the point in which you have too many options for radio buttons is arguable.
